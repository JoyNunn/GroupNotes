AWS is responsible for security of the cloud.

AWS handles the security of the physical infrastructure that hosts your resources. This

infrastructure includes the following:
Physical security of data centers: Physical fences, security cameras, card readers, hidden locations, disk degaussing and destruction, 2FA, etc.

Hardware Infrastructure: Physical servers, storage devices, etc.

Software Infrastructure: Operating systems, service applications, virtualization software, access logging, auditing

Network Infrastructure: Routers, switches, load balancers, firewalls, cabling, intrusion detection

Virtualization: Logical isolation of instances

Customer is responsible for security in the cloud.

Tasks that customers are responsible for according to the shared responsibility model

  • Selecting, upgrading, and patching instance OS
  • Securing application
  • Configuring security groups and firewalls (including intrusion detection and prevention)
  • Managing network configuration
  • Creating user accounts (users have entered an ID and password before use of application)

Customers retain complete control over their content. Customers are responsible for managing security requirements for their content, including:

  • Which content is stored
  • Which AWS services are used with the content
  • Which country the content is stored in
  • Format of content, and whether it is masked, anonymized, or encrypted
  • Who has access to that content and how those access rights are managed

AWS S3 (Simple Storage Service)

  • Data is stored as objects in buckets.
  • Object key: property that uniquely identifies an object in an S3 bucket
  • Storage is virtually unlimited, but one object has a limit of 5 GB. This limit raises to 40GB if you use a tool like Snowball.
  • S3 is designed for 11 9s (99.999999999) of availability.
  • Customers have granular access to buckets and objects.
  • *Granular access = dedicated level of access
  • Automatic redundancy (3 copies)  - files are stored in 3 separate availability zones.
  • Bucket names must be globally unique.

Object-level Storage Classes of S3

  • S3 Standard - Low latency, high throughput, frequent access. Expensive.
  • S3 Intelligent-Tiering - For when you can’t predict the access patterns of files. Moves objects down tiers if they aren’t accessed frequently, and up a tier once they’re accessed again.
  • S3 Standard-Infrequent Access (S3 Standard-IA)
  • S3 One Zone-Infrequent Access (S3 One Zone-IA)
  • Simple Storage Service Glacier - For long term storage of files (Cheaper)
  • Glacier Deep Archive - For very long term storage of files (Cheapest)

S3 Bucket and Object URL Structure:

  • Virtual-Hosted Style: https://<bucket>.<region>.s3.amazonaws.com/<key>
  • Path Style (Deprecated): https://<region>.s3.amazonaws.com/<bucket>/<key>
  • Updated Object URL structure article with information on exceptions, traps, and gotchas
  • Amazon blog post on Path Style deprecation

AMI Benefits:

Repeatability

Reusability

Recoverability

By default, you start with a bare operating system, but once you configure and upload things to a virtual machine, you can save that as an AMI. Then you can launch that configured machine as many times as you want, to whatever configurations of hardware that you want. An AMI is basically how you save your work in the cloud so it can be reproduced into a different machine anytime you want.

A starting AMI is analogous to installing a new operating system on a physical computer. Saving an AMI and then starting new instances from that is like creating a Recovery Point on a physical computer, and then restoring several machines to that point.

Instance Type:

This is analogous to picking out a processor, memory, graphics card, and hard drive for a physical computer. Depending on your workload, you may choose an instance type which is focused on CPU processing, RAM, GPU processing, or storage space. There are a wide variety of virtual configurations to choose from.

Network Adapters for EC2:

Elastic Fabric Adaptor (Fastest)
Elastic Network Adaptor
Intel Thingamabob (Slowest)

These settings determine how your virtual machine communicates with other services. High performance applications may benefit from very high-speed connections with other AWS services, such as machine learning, etc.

IAM:

Grants permission for Amazon services to access other Amazon services. Never put Amazon credentials onto an EC2 instance, this is bad security practice.

This setting allows your virtual machines to access other Amazon services, such as S3, databases, networking, and content delivery.

Instance Types:

On-Demand: Short, Spiky, Unpredictable workloads

Spot instances: Time insensitive workloads. Applications with flexible start and end times. Very low compute prices.

Reserved instances: Steady state workloads. Predictable workloads. Able to make upfront payments to reduce cost even further.

Dedicated Hosts: Highly sensitive workloads. Bring your own license. Best for following compliance and regulation.

Launching Instances via CLI:

aws ec2 run-instances --image-id ami-1a2b3c4d --count 1 --instance-type c3.large --key-name MyKeyPair --security-groups MySecurityGroup --region us-east-1

 aws specifies an invocation of the AWS CLI

ec2 Specifies an invocation of the Amazon EC2 service command

run-instances The subcommand that is being invoked
image-id This parameter is followed by an AMI ID. All AMIs have a unique AMI

ID.

count You can specify more than one.

instance-type You can specify the instance type to create (for example a c3.large

instance).

key-name In the example, assume that MyKeyPair already exists.

security-groups In this example, assume that MySecurityGroup already

exists.

region AMIs exist in an AWS Region, so you must specify the Region where the AWS CLI will find the AMI and launch the EC2 instance.

Command Syntax

man -i whoami

Command: man in this case. This is the main thing you are executing.

Option, Flag or Switch:-i in this case. Flags change the behavior of the command. Multiple flags can be put in sequence, or added onto a single dash: -l -a -h and -lah are equally valid.

Argument: "whoami" in this case. The argument tells the interpreter what to execute the command on, and how.

Computer Networking:

A computer network is a collection of computing devices (PCs, printers, anything) that are logically connected together to communicate and share resources.

Node: Any device on the network.

Host: A host is a node that has a unique function. Other devices connect to hosts so they can access data or services. Ex: Server

OSI (Open Systems Interconnection) Model: (Layers)

  1. Physical - literal physical components and electricity, radio waves, or other physical transmission media
  2. Data Link - MAC address
  3. Network - IP Address
  4. Transport - TCP (more reliable/slower)/UDP (less reliable/faster)
  5. Session - Establishes and keeps the session. (Port #)
  6. Presentation - Encryption and Compression, Encoding
  7. Application - Program to view the info. Web browser, ICQ, Zoom, etc.

Data flows from 7-1 in the client computer, then from 1-7 in the host computer.

At layer 2, a message or data is called a “Frame”. They are associated with a MAC address.

At layer 3, a message or data is called a “Packet”. They are associated with an IP address.

IPv4 addresses contain 8 bytes which is equal to 32 bits.

OSI Model for Network Troubleshooting Purposes:

The OSI (Open Systems Interconnection) model is a conceptual framework used to understand network interactions in 7 distinct layers.

Each layer has specific functions and communicates with the layers directly above and below it.

  1. Physical Layer
  • Function: This layer involves the physical connection between devices, including cables, switches, and the electrical signals transmitted
  • Troubleshooting Tips:
  • Check physical connections (cables, ports).
  • Use appropriate tools (cable testers) to identify wiring issues.
  • Ensure devices are powered on.
  1. Data Link Layer
  • Function: Responsible for node-to-node data transfer and error detection/correction. It includes protocols like Ethernet and Wi-Fi
  • Troubleshooting Tips:
  • Verify MAC addresses and ensure they’re configured correctly.
  • Check for collisions or errors in the network interface statistics.
  • Use tools like Wireshark to analyze Ethernet frames for issues.
  1. Network Layer
  • Function: Manages routing and forwarding of data packets. This layer includes protocols like IP (Internet Protocol)
  • Troubleshooting Tips:
  • Check IP address configurations (static vs. dynamic).
  • Use commands like ping to test connectivity between devices.
  • Verify routing tables and ensure proper routes are in place.
  1. Transport Layer
  • Function:  Ensures reliable data transfer, error recovery, and flow control, utilizing protocols like TCP and UDP.
  • Troubleshooting Tips:
  • Check for open/closed ports using tools like netstat.
  • Analyze application logs to identify issues with data transmission.
  • Use traceroute to identify where packets are getting dropped.
  1. Session Layer
  • Function:  Manages sessions between applications, including establishing, maintaining, and terminating connections.
  • Troubleshooting Tips:
  • Validate session management protocols (like RPC or NetBIOS).
  • Review application settings to ensure proper session handling.
  1. Presentation Layer
  • Function:  Translates data formats and encrypts/decrypts data for the application layer.
  • Troubleshooting Tips:
  • Ensure compatible data formats between sender and receiver.
  • Check for encryption issues that may prevent data from being read properly.
  1. Application Layer
  • Function: The closest layer to the end-user, it provides network services to applications (e.g., HTTP, FTP).
  • Verify application configurations and user permissions.
  • Check for application errors or service outages.
  • Use browser developer tools or API testing tools for web applications.

General Troubleshooting Steps

  1. Identify the Problem: Gather details like error messages or symptoms.
  2. Define the Scope: Check if the issue affects just one device or multiple devices/users.
  3. Use OSI Layers: Start troubleshooting from the lowest layer involved, and move up as needed.
  4. Isolate the Issue: Test each component separately to find the faulty part.
  5. Document Your Steps: Record what you've tried and the results to avoid repeating steps.

By using the OSI model, you can methodically solve network issues, pinpoint where they occur, and apply the right fixes for a quicker resolution.

CIDR Notation: 192.168.5.0/xx

Subnet Range: The above notation represents a “subnet range”. 192.168.5 is the subnet. The last octet of the IP address notates the device that’s assigned to that subnet. The numbers after the / determine how many bits are reserved for the network prefix.

0.0.0.0/0: Every possible IP address. The entire internet. Supports up to 4,294,967,296 hosts.

44.0.0.1/8: 44 is the network prefix, 0.0.1 is the device identifier. Class A Network. Supports up to 16,777,214 hosts.

128.16.0.1/16: 128.16 is the network prefix, 0.1 is the device identifier. Class B Network. Supports up to 65,534 hosts.

192.168.1.1/24: 192.168.1 is the network prefix, .1 is the device identifier. Supports up to 256 hosts.

A subnet mask is a 32-bit number created by setting host bits to all 0s and setting network bits to all 1s.

Class B: 255.255.0.0

Class C: 255.255.255.0

Types of Network Devices:

Switch: A switch is a device that connects the nodes of a network together. Every hardwired device on the network uses a NIC to connect to a port on the switch. Mainly uses Level 2 on the OSI model. Transmits only to the receiving device using a MAC address.

Router: A network device that connects multiple network segments together, or connects a network to the internet.

Modem: Connects a home or office to the internet. Connects to the internet port, and then to the router.

Types of Networks:

WAN: Wide area network. Over a wide geographical area.

LAN: Local area network. Over a limited geographical area.

The two most common models for LAN are:

Client-server: the data management and application hosting are centralized at the server, and distributed to the clients.If the server goes down, no client can access the network until the server is restored.

Examples:

  • File server and desktop clients
  • Print server and desktop clients

Peer-to-peer: each node has its own data and applications and is responsible for its own management and security. Peers are equally privileged participants in the architecture.

Conditions:

  • Users are responsible for backing up each node
  • Security requirements are not restrictive
  • A limited number of peers are used

A network protocol defines the rules for formatting and transmitting data between devices on a network

  • Typically operates at layer 3 (Network) and layer 4 (Transport) of the OSI model.

Connection oriented protocol: establishes a connection, waits for a response, creates session between sender and receiver, uses synchronous (real time) communication

Connectionless protocol: little overhead because it does not check if a session is established, does not require a session between sender and receiver, uses asynchronous communication

Examples of Network Protocols:

TCP/IP, connection-oriented protocol, data sent via this protocol is broken down into smaller chunks called packets. The goal of TCP/IP was to support an interconnection of networks (internet)

TCP, great for transferring important files since there is a guarantee of connection, even though there is a larger overhead (time)

  • TCP handshake has 3 messages
  • Synchronize (SYN)
  • Synchronize/Acknowledge (SYN/ACK)
  • Acknowledge (ACK)

UDP, connectionless communication and no guarantee of delivery or ordering of data, based on speed, does not ensure data delivery

  • E.g. media, in movies there are times when you might see buffering and things may seem off or out of order

Internet Protocols (IP) - a network protocol that establishes the rules for relaying and routing data in the internet

  • uses IP addresses to identify devices
  • uses port numbers to identify endpoints
  • supports subnetting to subdivide a network

IP addresses

  • Uniquely identifies a device on a network
  • Each device on a network has an IP address with 2 functions
  • Identifies host and network
  • Used for location addressing
  • Work at layer 3 of OSI model
  • Can be assigned in dynamic (where assigned IP address does not change) or static (when assigned IP does not change)
  • Can be public or private

Example issues with IP addresses (layer 3)

  • Latency: site or app takes a long time to load, e.g. time outs
  • Figuring out where and what causes latency in corporate settings is important because it saves time and money
  • Traceroute is a command that reports on the path
  • Unresponsive server
  • Dynamic assigned IP address that should be static

Troubleshooting commands that can be used for layer 3

  • Ping 
  • Traceroute

Subnetting Basics

Subnet (Subnetwork): A smaller network created from a larger network to efficiently manage IP addresses

Parts of a Subnet

  • Network Portion - identifies the specific network
  • Host Portion - identifies a specific device (host) within that network

Purpose of Subnetting

  • Improves management of IP addresses
  • Enhances security by isolating network segments
  • Optimizes performances by reducing network congestion

CIDR (Classless Inter-Domain Routing) Notation: Specifies IP address ranges and subnet masks

  • Format: An IP address followed by a / and a number (e.g., 192.168.1.0/24)
  • The number after the slash / (e.g., /24) indicates the bits used for the network portion of the address
  • 192.168.1.0/24 means that the first 24 bits are used for the network, and the remaining 8 bits are used for host addresses.

Subnet Mask: A 32-bit number dividing an IP address into network and host portions.

  • Identifies which part of the IP address is for the network and which is for the host.
  • Determines the size of the subnet and available IP addresses.
  • Common Subnet Masks:
  • /24 (255.255.255.0): 256 IP addresses (254 usable).
  • /16 (255.255.0.0): 65,536 IP addresses (65,534 usable).
  • /8 (255.0.0.0): Over 16 million IP addresses (16,777,214 usable).
